🔒 Segurança

CRÍTICO

Chave SERVICE_ROLE_KEY usada diretamente em página server-side

A página /orcamento/[token]/page.tsx instancia createServerClient com a chave SUPABASE_SERVICE_ROLE_KEY diretamente no componente. Embora rode no servidor, isso significa que a chave de administrador do banco está espalhada no código da aplicação. Se houver qualquer vazamento de erro SSR ou log, a key pode ser exposta.

Impacto no negóbio: Um invasor com acesso à key pode ler, alterar ou apagar TODOS os dados — clientes, pedidos, orçamentos, tudo.

✅ Correção: Criar uma API route (app/api/orcamento/[token]/route.ts) que encapsula a consulta. A página chama a API. A key fica isolada no servidor.
📄 src/app/orcamento/[token]/page.tsx:12-21
IMPORTANTE

6 erros silenciosos — código engole exceções sem avisar

Existem 6 pontos no código onde erros são capturados e completamente ignorados (catch {}). Se algo der errado — falha no banco, erro ao gerar PDF, falha ao salvar assinatura — o app simplesmente para de funcionar sem mostrar nenhuma mensagem.

Impacto no negócio: Você pode perder dados que o cliente achou que salvou. Um orçamento que não gerou PDF, um cliente que não foi salvo, uma assinatura que desapareceu.

✅ Correção: Substituir todos os catch {} por catch (err) { console.error('[contexto]', err); } e mostrar mensagem amigável ao usuário.
📄 PedidoForm.tsx:229,739 · orcamentos/novo/page.tsx:105 · pedidos/novo/page.tsx:52 · orcamentos/[id]/page.tsx:81 · server.ts:20
IMPORTANTE

Vulnerabilidade conhecida em dependência (PostCSS XSS)

A biblioteca PostCSS (usada pelo Next.js) tem uma vulnerabilidade de segurança moderada — XSS via CSS (CVE GHSA-qx2v-qp2m-jg93, CVSS 6.1). O npm audit encontrou 2 problemas.

Impacto no negócio: Um atacante poderia injetar código malicioso através de CSS manipulado.

✅ Correção: Rodar npm update next para atualizar para versão com PostCSS >= 8.5.10.
📦 postcss < 8.5.10 (via Next.js)

📝 Qualidade de Código

CRÍTICO

PedidoForm.tsx — componente monolítico de 745 linhas

O arquivo PedidoForm.tsx tem 745 linhas e 14 hooks (useState, useEffect, useCallback, useRef). Ele cuida de: dados do cliente, busca CEP, produtos, medidas, cálculos de m², calha, assinaturas, pagamento, máscaras de input, upload de fotos, e muito mais — tudo em um único componente.

Impacto no negócio: Cada nova feature ou correção nesse arquivo é arriscada. Pode quebrar algo que estava funcionando. Um desenvolvedor novo leva horas para entender o código.

✅ Correção: Dividir em componentes menores:
• ClientInfoSection.tsx — dados do cliente + busca CEP
• ProductSection.tsx — lista de produtos + autocomplete
• ProductMeasure.tsx — medidas individuais
• PaymentSection.tsx — forma de pagamento
• SignatureSection.tsx — assinaturas
• PedidoForm.tsx — orquestrador (~150 linhas)
📄 src/components/pedido/PedidoForm.tsx (745 linhas, 14 hooks)
IMPORTANTE

~25 usos de "any" no TypeScript — tipagem desativada

O código tem aproximadamente 25 usos de any, principalmente na página de agendamentos (18 só nela). O any desativa completamente a verificação de tipos do TypeScript — é como escrever JavaScript puro, perdendo toda a segurança que o TypeScript oferece.

Impacto no negócio: Erros que seriam detectados automaticamente pelo TypeScript passam despercebidos e só aparecem quando o cliente já está usando o app.

✅ Correção: Criar interfaces TypeScript para: Appointment, DocumentSummary, DeadlineOrder, ClientWithDocs. Substituir todos os any.
📄 agendamentos/page.tsx (18) · clientes/[id]/page.tsx (2) · dashboard/page.tsx (1) · PedidoForm.tsx (2) · orcamentos/novo (2)

Auth layout como client component

O layout das páginas protegidas (src/app/(auth)/layout.tsx) é um client component que redireciona via useEffect. Isso significa que o conteúdo protegido é enviado ao navegador antes do redirecionamento.

✅ Melhoria: Mover a verificação de auth para Server Components, usando redirect() do Next.js diretamente.
📄 src/app/(auth)/layout.tsx
SUGESTÃO

Pasta /lib/store existe mas está vazia

A pasta src/lib/store existe no projeto mas não contém nenhum arquivo. Isso indica que foi criada para um gerenciamento de estado (como Zustand) que nunca foi implementado. O auth usa um sistema manual de estado global com listeners.

✅ Melhoria: Remover a pasta vazia ou implementar um gerenciamento de estado adequado (Zustand ou Jotai).

⚡ Performance

IMPORTANTE

11 consultas ao banco com select('*') — dados desnecessários

Foram encontradas 11 consultas usando select('*') ou sem especificar colunas. Isso traz TODOS os dados da tabela, inclusive colunas pesadas que não são usadas, deixando o app mais lento conforme os dados crescem.

Impacto no negócio: O app vai ficando mais lento conforme você acumula pedidos e clientes. Páginas como a lista de clientes e o dashboard são as mais afetadas.

✅ Correção: Especificar só as colunas necessárias. Ex: select('id, name, phone') em vez de select('*').

Imagens sem otimização — sem next/image

As fotos dos pedidos são exibidas usando <img> nativo em vez do <Image> do Next.js. O Next.js otimiza imagens automaticamente (redimensionamento, formato WebP, lazy loading).

✅ Melhoria: Usar next/image com sizes e priority={false}.
