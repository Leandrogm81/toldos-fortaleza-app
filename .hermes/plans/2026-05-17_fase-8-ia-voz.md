## 1. Objetivo

Adicionar duas features de automação com IA:

- **IA de preenchimento** — colar conversa do WhatsApp → IA extrai dados e preenche o formulário
- **Reconhecimento de voz** — ditar campos do pedido pelo microfone

---

## 2. Funcionalidades

### 2.1 IA de Preenchimento (Text-to-Form)

**Contexto do usuário:**
O Leandro recebe pedidos por WhatsApp. O cliente manda algo como:

> "Quero um toldo de policarbonato 5,60m x 1,40m na minha casa. Meu nome é João Silva, telefone 11 99999-8888, moro na Rua das Flores, 123, Bairro Jardim. Pode ser instalado em 20 dias?"

**Fluxo:**
1. Usuário cola o texto da conversa do WhatsApp numa caixa de texto (ou digita)
2. Clica em "Analisar com IA"
3. A IA extrai e preenche automaticamente:
   - Nome do cliente
   - Telefone
   - Endereço (se mencionado)
   - Tipo de produto (toldo, cobertura, cortina...)
   - Material (policarbonato, lona...)
   - Medidas (largura × comprimento)
   - Prazo desejado
   - Observações
4. Usuário revisa e ajusta antes de salvar

**Tecnologia:**
- API do OpenRouter (modelos baratos: DeepSeek V4 Flash ou similar)
- Prompt estruturado pedindo JSON com os campos extraídos
- Botão "Preencher formulário" que popula o `PedidoForm`

**Implementação:**
- API Route no Next.js (`/api/ai/extract`) que chama OpenRouter
- Frontend: componente `AiFillButton` com textarea + botão "Analisar"
- Feedback visual: loading spinner + preview dos campos extraídos
- Fallback: se a IA não entender algo, destaca em amarelo

### 2.2 Reconhecimento de Voz (Speech-to-Text)

**Fluxo:**
1. Usuário clica no ícone de microfone em qualquer campo do formulário
2. Navegador captura áudio (Web Speech API)
3. Texto transcrito aparece no campo
4. Funciona offline? Não — Web Speech API depende de servidor Google (Chrome) ou Apple (Safari)

**Implementação:**
- Hook `useSpeechRecognition` usando `window.SpeechRecognition` ou `webkitSpeechRecognition`
- Botão microfone com animação de "ouvindo" (pulsação vermelha)
- Suporte a português (`lang: 'pt-BR'`)
- Fallback: se navegador não suportar, esconde o botão

**Limitações:**
- Chrome Android: ✅ Funciona bem
- Safari iOS: ✅ Funciona (desde iOS 14.5)
- Firefox: ❌ Não suporta
- Precisão: ~90% para português claro

## 3. Arquitetura — API Route

```
POST /api/ai/extract
Body: { text: "string da conversa do WhatsApp" }
Response: { name, phone, address, product, material, width, height, ... }
```

**Prompt (exemplo):**
```
Extraia os seguintes dados do texto abaixo em JSON. Se não encontrar, deixe null.

Texto: "{text}"

Campos: name, phone, address, neighborhood, city, product_type (toldo_fixo|cobertura_retratil|cortina|outro), material (policarbonato|lona|outro), width_m (número), height_m (número), install_days (número), notes

Responda SOMENTE o JSON, sem texto adicional.
```

---

## 3. Riscos

| Risco | Mitigação |
|-------|-----------|
| IA extrair dados errados | Sempre mostrar preview antes de preencher. Usuário revisa. |
| Custo da API OpenRouter | DeepSeek V4 Flash custa ~$0.14/1M tokens. ~100 extrações/mês = centavos. |
| Web Speech API não disponível | Esconder botão de microfone se não suportado. |
| Latência da IA | Mostrar skeleton/spinner durante a análise (1-3 segundos). |
